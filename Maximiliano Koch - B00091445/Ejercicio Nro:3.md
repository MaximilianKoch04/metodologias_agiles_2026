# Ejercicio 3
## Enunciado
Qué ventajas y desventajas tienen los diagramas UML ?
- Ayuda: Se puede relacionar con los plano de la Ingeniería tradicional


## Resolución
Ventajas:
- Sirven para intentar estructurar y modelar el comportamiento del sistema. Por ejemplo, enfoques como el proceso ICONIX utilizan una cantidad mínima y racionalizada de diagramas UML (casos de uso, robustez, secuencia y clases) para guiar el paso a paso desde el requerimiento hasta el código.
Desventajas:
- Son poco expresivos para el usuario final: un cliente no puede validar si el software realmente le sirve o resuelve su problema a partir de un diagrama UML; necesita usar el producto funcionando .
- Son poco expresivos para los desarrolladores: los diagramas no logran especificar muchos detalles técnicos importantes que recién aparecen en el momento de programar .
- El problema de la "Invisibilidad": en la construcción tradicional, un plano detalla todo el edificio de forma clara. En el software, intentar representar toda su estructura con diagramas genera grafos superpuestos muy complejos que dificultan el análisis y la comunicación entre las personas. Por este motivo, la metáfora de planificar el software con "planos detallados" no funciona.
